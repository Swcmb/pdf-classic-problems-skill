# pdf-classic-problems Skill

> 将图片组成的 PDF 自动处理为两个 Markdown 文档：`经典题列表.md`（仅含题目列表）与 `经典题详解与拓展.md`（仅含详解与拓展）。
> 适用于理工科教材/习题集扫描件（纯图片型 PDF）。

## 1. 概述与触发条件

### 1.1 触发词

`图片PDF`、`扫描件PDF`、`经典题`、`经典题详解`

### 1.2 适用场景

- 理工科教材/习题集扫描件（纯图片型 PDF）
- 需要从 PDF 中筛选经典题并生成详解的场景

### 1.3 不适用场景

- 文本型 PDF（使用 pdf Skill 处理）
- 非学术 PDF（如小说、报告）
- 混合型 PDF（部分页为文本层、部分页为扫描图片）—— **不支持，作为非目标声明**
- 加密或受密码保护的 PDF

## 2. 前置条件检查（阶段 1）

### 2.1 执行环境检查

运行环境检查脚本：

```bash
python3 skill/scripts/check_env.py <pdf_path>
```

### 2.2 解析检查结果

读取脚本返回的 JSON 状态报告：

- 若 `status == "ok"`：环境就绪，进入阶段 2
- 若 `status == "error"`：向用户输出 `install_commands` 中的安装命令，**终止流程**

### 2.3 加密检测

若 JSON 中 `pdf_file.encrypted == true`：
- 向用户提示"PDF 受密码保护，请解除密码保护后重试"
- **终止流程**

## 3. OCR 提取流程（阶段 2）

### 3.1 执行 OCR 提取

运行 OCR 提取脚本：

```bash
python3 skill/scripts/ocr_extract.py <pdf_path> <output_dir> [--dpi 300] [--lang chi_sim+eng] [--batch-size 20]
```

### 3.2 检查提取质量

读取脚本返回的 JSON 统计报告，检查 `quality_warning` 字段：

- 若 `quality_warning == false`：OCR 质量正常，进入阶段 3
- 若 `quality_warning == true`：
  1. 警告用户 OCR 质量可能不佳
  2. 输出 `empty_pages` 列表与 `avg_text_length` 值
  3. 建议尝试 `--dpi 400` 重新提取
  4. 询问用户是否继续（继续则 LLM 尽力处理）

### 3.3 空 PDF 检测

若 `total_pages == 0` 且退出码为 1：
- 向用户提示"PDF 无有效页面，请检查文件是否为空或损坏"
- **终止流程**

## 4. 题目识别规则（阶段 3）

### 4.1 读取页面文本

读取 `<output_dir>/pages/` 目录下的所有 `page_XXXX.txt` 文件，按页码顺序合并。

### 4.2 识别题目

参照 `skill/references/problem-coding.md`，执行以下操作：

1. 识别章节标题（中文数字 / 阿拉伯数字模式）
2. 识别题号（例题 / 习题 / 定理证明 / 定义推导）
3. 应用误匹配过滤规则（位置约束 / 内容约束 / 上下文约束）
4. 判定题目边界（起点 / 终点 / 跨页 / 公式完整性）
5. 生成题目编码（`章节号-题型序号` 格式）

### 4.3 生成题目索引

生成内部题目索引（不输出文件），包含：
- 题号编码
- 所属章节
- 题型
- 题目内容摘要

### 4.4 空索引检测

若题目索引长度为 0：
- 输出 OCR 原始文本前 5 页供用户检查
- **终止流程**

## 5. 经典题筛选流程（阶段 4）

### 5.1 评分

参照 `skill/references/selection-criteria.md`，对每道题按三大标准评分：

| 标准 | 权重 |
|------|------|
| 核心概念代表性 | 40% |
| 方法论启发性 | 35% |
| 典型性与易错性 | 25% |

每项标准评分 1-5 分，≥ 3 分入选候选池。

### 5.2 计算综合评分

```
总分 = 核心概念 × 0.40 + 方法论 × 0.35 + 典型性 × 0.25
```

入选阈值：总分 ≥ 3.5

### 5.3 筛选

1. 按总分降序排列
2. 选取前 N 题（默认 10 题，可配置 8-12 题）
3. 确保覆盖各主要章节（每章至少 1 题，如有）
4. 若同章节多题高分，取最高分者

### 5.4 数量不足降级

| 总题数 | 策略 |
|--------|------|
| < 8 | 选取总题数的 50-80%，提示用户 |
| < 3 | 提示 PDF 可能不含足够题目 |
| = 0 | 转入异常处理（4.4 空索引检测） |

## 6. 详解生成规范（阶段 5）

### 6.1 生成详解内容

参照 `skill/references/output-format.md`，为每道经典题生成四部分内容：

1. **题目**：原题内容，保留公式与格式
2. **解答**：完整解题过程，逐步推导
3. **分析**：解题思路说明，关键步骤解释
4. **反思与拓展**：方法总结 + 至少 1 道拓展题（解答使用 `<details>` 折叠）

### 6.2 公式格式

| 场景 | 格式 |
|------|------|
| 行内公式 | `$...$` |
| 行间公式 | `$$...$$` |
| 多行公式 | `\begin{align}...\end{align}` |
| 分段函数 | `\begin{cases}...\end{cases}` |

### 6.3 断点续传机制

详解生成过程中，采用断点续传机制防止中断丢失进度：

1. 每完成一道题的详解后，将题号编码写入 `<output_dir>/.progress` 文件
2. `.progress` 文件格式：每行一个题号编码（如 `10.1.1-EX1`），UTF-8 编码，无 BOM
3. 若生成中途中断，下次运行时：
   - 读取 `.progress` 文件获取已完成的题号
   - 跳过已生成题目，从断点继续
4. 若 `.progress` 不存在（首次失败），扫描 `经典题详解与拓展.md` 中已有的 `## {题号}` 标题，与计划生成列表比对，找出缺失题目并从头补生成
5. 确保已生成题目的完整性（四部分齐全）

## 7. 输出与清理（阶段 5）

### 7.1 输出文件

- 输出路径：默认 `/workspace/`，可通过参数配置
- 输出目录不存在时自动创建
- 输出文件已存在时覆盖写入
- 文件名：`经典题列表.md`、`经典题详解与拓展.md`

### 7.2 经典题列表.md

按 `skill/references/output-format.md` 第 1 节格式输出：
- 按章节号升序排列
- 同章节内按题号顺序排列
- 仅含题号，不含题目内容

### 7.3 经典题详解与拓展.md

按 `skill/references/output-format.md` 第 2 节格式输出：
- 每题包含四部分（题目/解答/分析/反思与拓展）
- 题间使用 `---` 分隔线
- 拓展题解答使用 `<details>` 折叠

### 7.4 清理

1. 删除临时目录（OCR 页面文本文件 `<output_dir>/pages/`）
2. 详解生成全部完成后删除 `.progress` 文件，避免陈旧记录影响后续运行

## 8. 异常处理指引

LLM 在各阶段执行时参照以下异常处理矩阵：

| 异常场景 | 所处阶段 | 检测方式 | 处理策略 |
|----------|----------|----------|----------|
| 环境检查失败 | 阶段 1 | `check_env.py` 返回 `status: "error"` | 输出安装命令，终止 |
| OCR 质量低 | 阶段 2 | `quality_warning == true` | 警告并建议调高 DPI，询问是否继续 |
| 题目识别为空 | 阶段 3 | 题目索引长度为 0 | 输出前 5 页 OCR 文本供检查，终止 |
| 选题数量不足 | 阶段 4 | 总题数 < 8 | 选取 50-80%，提示用户 |
| 详解生成中途失败 | 阶段 5 | LLM 上下文溢出或异常 | 从 `.progress` 断点续传 |
| PDF 加密 | 阶段 1 | `check_env.py` 返回 `encrypted: true` | 提示解除密码保护，终止 |
| 输出目录不可写 | 阶段 5 | `os.makedirs` 或文件写入失败 | 提示检查权限或更换目录 |
| OCR 乱码（非空但质量差） | 阶段 2 | `avg_text_length < 50` | 触发 `quality_warning`，同 OCR 质量低处理 |
| 空 PDF（total_pages==0） | 阶段 2 | `ocr_extract.py` 返回 `total_pages==0` 且退出码 1 | 提示用户 PDF 无有效页面，终止 |
